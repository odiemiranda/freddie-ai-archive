# AAR-0005: Staged Proposals Storage + brain.db Integration

## Objective

Create the storage layer for soul change proposals that fail validation gates or the judge panel. Staged proposals are markdown files in `vault/studio/memory/{agent}/staged/` with rich frontmatter. They are indexed in brain.db (FTS5 + embeddings) and searchable via `--search-staged` or included in `--recall-all` results.

## Context

When the autonomous pipeline rejects a soul change (gate failure or judge dissent), the proposal must be persisted for human review. The morning briefing (AAR-0006) reads these files to display pending changes.

**Directory convention:**
```
vault/studio/memory/{agent}/staged/
  ├── 20260407-143022-boundary-first-instinct.md
  └── 20260408-091500-always-triangulate.md
```

These files are NOT part of the inbox/knowledge/archive lifecycle. They are a separate tier: pending human decisions about soul identity.

#### Write Function

```typescript
import { existsSync, mkdirSync, writeFileSync, readdirSync, readFileSync } from "fs";
import { join } from "path";
import { fromRoot } from "./paths.js";
import dayjs from "./dayjs.js";

const MEMORY_ROOT = fromRoot("vault", "studio", "memory");

/**
 * Write a staged proposal to disk.
 * 
 * @param proposal - The rejected soul change proposal
 * @returns The relative path to the written file (relative to vault/studio/)
 */
export function writeStagedProposal(proposal: StagedProposal): string {
  const stagedDir = join(MEMORY_ROOT, proposal.agent, "staged");
  mkdirSync(stagedDir, { recursive: true });

  const now = dayjs();
  const timestamp = now.format("YYYYMMDD-HHmmss");
  const slug = proposal.proposedChange
    .slice(0, 60)
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, "-")
    .replace(/^-|-$/g, "")
    .slice(0, 40);

  const filename = `${timestamp}-${slug}.md`;
  const filePath = join(stagedDir, filename);

  const content = renderStagedProposal(proposal, now);
  writeFileSync(filePath, content, "utf-8");

  // Return relative path for brain.db indexing
  return `memory/${proposal.agent}/staged/${filename}`;
}
```

#### Render Function

```typescript
function renderStagedProposal(
  proposal: StagedProposal,
  now: ReturnType<typeof dayjs>
): string {
  const gatesSummary = proposal.gateResults.results
    .map(r => `${r.gate}: ${r.pass ? "PASS" : "FAIL"} — ${r.reason}`)
    .join("\n  ");

  let judgesSummary = "Panel did not run (gates failed)";
  if (proposal.judgeResults) {
    judgesSummary = proposal.judgeResults.votes
      .map(v => `${v.model}: ${v.vote.toUpperCase()} — ${v.reason}`)
      .join("\n  ");
  }

  return `---
type: staged-proposal
agent: ${proposal.agent}
date: ${proposal.date}
section: "${proposal.section}"
change_type: ${proposal.changeType}
status: ${proposal.status}
reason: "${proposal.reason.replace(/"/g, '\\"')}"
created: ${now.format("YYYY-MM-DDTHH:mm:ss")}
---

# Staged Soul Change: ${proposal.agent}

## Proposed Change

**Section:** ${proposal.section}
**Type:** ${proposal.changeType}

${proposal.proposedChange}

## Evidence

${proposal.evidence}

## Gate Results

  ${gatesSummary}

## Judge Results

  ${judgesSummary}

## Reason for Staging

${proposal.reason}

## Decision

Status: **${proposal.status}**

<!-- To approve: change status to "approved" in frontmatter -->
<!-- To reject: change status to "rejected" in frontmatter -->
`;
}
```

#### Read Functions

```typescript
/**
 * List all staged proposals for an agent (or all agents).
 */
export function listStagedProposals(agent?: string): StagedFile[] {
  const results: StagedFile[] = [];
  const agents = agent ? [agent] : getAllAgentDirs();

  for (const a of agents) {
    const stagedDir = join(MEMORY_ROOT, a, "staged");
    if (!existsSync(stagedDir)) continue;

    const files = readdirSync(stagedDir).filter(f => f.endsWith(".md"));
    for (const f of files) {
      const content = readFileSync(join(stagedDir, f), "utf-8");
      const fm = parseStagedFrontmatter(content);
      if (fm) {
        results.push({
          path: `memory/${a}/staged/${f}`,
          agent: a,
          filename: f,
          proposal: {
            agent: a,
            date: fm.date ?? "",
            proposedChange: extractSection(content, "Proposed Change") ?? "",
            section: fm.section ?? "",
            changeType: fm.change_type ?? "add",
            evidence: extractSection(content, "Evidence") ?? "",
            gateResults: { passed: false, results: [], failedGates: [] },  // simplified for listing
            judgeResults: null,
            reason: fm.reason ?? "",
            status: fm.status ?? "pending",
          },
        });
      }
    }
  }

  return results;
}

/**
 * Get pending (unreviewed) staged proposals for an agent.
 */
export function getPendingProposals(agent?: string): StagedFile[] {
  return listStagedProposals(agent).filter(s => s.proposal.status === "pending");
}

// Helper: get all agent directories under memory/
function getAllAgentDirs(): string[] {
  if (!existsSync(MEMORY_ROOT)) return [];
  return readdirSync(MEMORY_ROOT, { withFileTypes: true })
    .filter(d => d.isDirectory() && d.name !== "raw")
    .map(d => d.name);
}

// Helper: parse frontmatter from staged file
function parseStagedFrontmatter(content: string): Record<string, any> | null {
  const match = content.match(/^---\n([\s\S]*?)\n---/);
  if (!match) return null;
  const fm: Record<string, any> = {};
  for (const line of match[1].split("\n")) {
    const m = line.match(/^(\w+):\s*"?(.*?)"?\s*$/);
    if (m) fm[m[1]] = m[2];
  }
  return fm;
}

// Helper: extract a ## section's content
function extractSection(content: string, sectionName: string): string | null {
  const regex = new RegExp(`## ${sectionName}\\n([\\s\\S]*?)(?=\\n## |$)`);
  const match = content.match(regex);
  return match ? match[1].trim() : null;
}
```

Add a sync function for staged proposals. Follow the pattern of `syncMemories()`:

```typescript
/**
 * Sync staged proposal files into brain.db.
 * Scans vault/studio/memory/*/staged/ directories.
 */
export async function syncStaged(opts?: { full?: boolean }): Promise<number> {
  const raw = getRawDb();
  let changed = 0;

  if (opts?.full) {
    raw.exec("DELETE FROM staged_proposals");
    raw.exec("DELETE FROM sync_meta WHERE table_name = 'staged_proposals'");
  }

  const currentPaths = new Set<string>();

  // Scan all agent staged directories
  if (!existsSync(MEMORY_ROOT)) return 0;
  const agentDirs = readdirSync(MEMORY_ROOT, { withFileTypes: true })
    .filter(d => d.isDirectory() && d.name !== "raw");

  for (const agentDir of agentDirs) {
    const stagedDir = join(MEMORY_ROOT, agentDir.name, "staged");
    if (!existsSync(stagedDir)) continue;

    const files = findMdFiles(stagedDir);
    for (const filePath of files) {
      const sourcePath = relative(VAULT_STUDIO, filePath).replace(/\\/g, "/");
      currentPaths.add(sourcePath);

      const rawContent = readFileSync(filePath, "utf-8");
      const hash = hashFile(rawContent);

      // Check if already synced
      const db = getDb();
      const existing = db.select().from(syncMeta).where(eq(syncMeta.sourcePath, sourcePath)).get();
      if (existing && existing.fileHash === hash && !opts?.full) continue;

      // Parse frontmatter
      const parsed = parseFile(filePath);
      const fm = parsed.frontmatter;
      const ts = now();

      raw.exec(`
        INSERT INTO staged_proposals (agent, section, change_type, proposed_change, reason, status, source_path, file_hash, created_at, updated_at)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
        ON CONFLICT(source_path) DO UPDATE SET
          agent=excluded.agent, section=excluded.section, change_type=excluded.change_type,
          proposed_change=excluded.proposed_change, reason=excluded.reason,
          status=excluded.status, file_hash=excluded.file_hash, updated_at=excluded.updated_at
      `.trim(), [
        fm.agent ?? agentDir.name,
        fm.section ?? "",
        fm.change_type ?? "add",
        parsed.body.slice(0, 2000),  // truncate for DB
        fm.reason ?? "",
        fm.status ?? "pending",
        sourcePath, hash, ts, ts,
      ] as any);

      // Upsert sync meta
      raw.exec(`
        INSERT INTO sync_meta (source_path, file_hash, table_name, synced_at)
        VALUES (?, ?, 'staged_proposals', ?)
        ON CONFLICT(source_path) DO UPDATE SET
          file_hash=excluded.file_hash, synced_at=excluded.synced_at
      `.trim(), [sourcePath, hash, ts] as any);

      changed++;
    }
  }

  // Remove orphans
  const allRows = raw.prepare("SELECT source_path FROM staged_proposals").all() as { source_path: string }[];
  for (const row of allRows) {
    if (!currentPaths.has(row.source_path)) {
      raw.exec("DELETE FROM staged_proposals WHERE source_path = ?", [row.source_path] as any);
      raw.exec("DELETE FROM sync_meta WHERE source_path = ?", [row.source_path] as any);
    }
  }

  return changed;
}
```

```typescript
/** Full-text search across staged proposals. */
export function searchStaged(query: string, opts?: { agent?: string; status?: string; limit?: number }): Array<{
  id: number; agent: string; section: string; changeType: string;
  proposedChange: string; reason: string; status: string;
  sourcePath: string; rank: number;
}> {
  const raw = getRawDb();
  const limit = opts?.limit ?? 20;
  const safeQuery = sanitizeFtsQuery(query);

  let sql = `
    SELECT s.id, s.agent, s.section, s.change_type, s.proposed_change,
           s.reason, s.status, s.source_path, rank
    FROM staged_fts
    JOIN staged_proposals s ON s.id = staged_fts.rowid
    WHERE staged_fts MATCH ?
  `;
  const params: any[] = [safeQuery];

  if (opts?.agent) {
    sql += " AND s.agent = ?";
    params.push(opts.agent);
  }
  if (opts?.status) {
    sql += " AND s.status = ?";
    params.push(opts.status);
  }

  sql += " ORDER BY rank LIMIT ?";
  params.push(limit);

  const rows = raw.prepare(sql).all(...params) as any[];
  return rows.map(r => ({
    id: r.id,
    agent: r.agent,
    section: r.section,
    changeType: r.change_type,
    proposedChange: r.proposed_change,
    reason: r.reason,
    status: r.status,
    sourcePath: r.source_path,
    rank: r.rank,
  }));
}
```

## Acceptance Criteria

1. **`writeStagedProposal()` creates files.** Files appear in `vault/studio/memory/{agent}/staged/` with correct frontmatter and content.
2. **Files are valid markdown.** Frontmatter parses correctly. Sections are readable.
3. **`listStagedProposals()` finds files.** Returns all staged files for an agent or all agents.
4. **`getPendingProposals()` filters by status.** Returns only status=pending files.
5. **`staged_proposals` table exists.** Created by `initBrain()`. Indexed by agent and status.
6. **FTS5 on staged proposals.** `staged_fts` virtual table exists with sync triggers.
7. **`syncStaged()` works.** Syncs staged files to brain.db. Hash-based incremental.
8. **`searchStaged()` works.** FTS5 search returns matching staged proposals.
9. **`--search-staged` flag works.** `bun run tool brain --search-staged "boundary"` returns results.
10. **`--recall-all` includes staged.** Cross-store search returns staged proposals alongside memories, decisions, etc.
11. **`--check` syncs staged.** `bun run tool brain --check` picks up new/changed staged files.

## Constraints

- Staged files are NOT part of the inbox/knowledge/archive lifecycle. `consolidate-memory.ts` must NOT process them. Verify that the consolidation tool's directory scanning skips `staged/`.
- The sync function follows the same pattern as `syncMemories()` — hash-based, incremental, orphan removal.
- Frontmatter parsing uses the simple regex approach (consistent with how other memory files are parsed). Do not add a YAML parser dependency.
- The `staged/` directory is created on-demand by `writeStagedProposal()`. It does not need to pre-exist.

## Out of Scope

- UI for approving/rejecting staged proposals (human edits the frontmatter directly in Obsidian)
- Vector embeddings for staged proposals (FTS5 is sufficient for this use case)
- Automatic cleanup of approved/rejected staged files
- Notifications when a staged proposal is approved/rejected
