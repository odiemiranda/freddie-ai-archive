# When using a 7B model for binary classification, what prompt design patterns improve output reliability and reduce 'flipping'? Few-shot examples? Structured output via GBNF grammar constraints? Temperature 0? Label ordering? What are the known failure modes to avoid? Include specific examples of prompt templates that work well for CONTRADICTION vs COMPATIBLE style classification.

**Temperature 0 and GBNF Grammar Constraints**
Setting the model **temperature to 0 is highly recommended when correctness is paramount**, as it forces the model to select the most likely token, yielding near-deterministic and replicable outputs [1, 2]. To guarantee strict adherence to binary labels and eliminate conversational "fluff" common in RLHF-trained models, **utilize Grammar-Based Network Format (GBNF) logit shaping** [3, 4]. GBNF operates at the API/inference layer to restrict the model's vocabulary during token sampling, ensuring it can *only* output your desired schema (e.g., exactly `CONTRADICTION` or `COMPATIBLE`, or a strictly formatted JSON) without dropping to raw logit distributions [4, 5].

**Few-Shot Examples and Chain-of-Thought (CoT)**
Empirical studies on requirements classification show that **Chain-of-Thought (CoT) combined with few-shot prompting is the most effective strategy** [1, 6]. However, few-shot learning in small large language models (sLLMs) is highly unstable, with accuracy swinging up to 14% based purely on which examples are selected [7]. To maximize reliability:
*   **Use Difficult Examples:** Counterintuitively, models perform significantly better (up to 10% higher accuracy) when provided with *difficult* few-shot examples rather than easy or random ones, as difficult examples provide richer contextual features for the model to learn the decision boundary [8].
*   **Force Reasoning First:** Structure the prompt so the model outputs a step-by-step reasoning preamble *before* outputting the final binary label [6]. 
*   **Repeat Instructions:** Bookend the prompt by placing task instructions and domain definitions at the top, and repeating the strict output formatting instructions at the very end [9].

**Known Failure Modes to Avoid**
*   **Positional Bias and Option Shuffling:** 7B/8B models are highly vulnerable to positional bias. For example, Llama-3-8B exhibits a severe intrinsic bias toward selecting the second option, with its stability dropping to 44%–52% when the order of options is merely swapped [10]. **Avoid presenting the task as a multiple-choice "A or B" selection; instead, ask the model to output the semantic label directly.**
*   **Spurious Few-Shot Patterns:** If your few-shot examples follow a hidden pattern (e.g., grouping all `COMPATIBLE` examples first, then all `CONTRADICTION` examples, or alternating them perfectly), the model will latch onto this spurious pattern and predict the next label in the sequence rather than actually classifying the text [11-13].
*   **Instruction Positional Sensitivity:** Small models are exceptionally sensitive to where instructions are placed in the prompt. Altering the order of instructions can cause up to a 20-point drop in robustness [14, 15]. Use a fixed, standard template across all pipeline runs.

**Optimized Prompt Template Structure**
Based on the most effective patterns for classification tasks [6, 9], here is a template optimized for a 7B model using CoT and strict formatting:

**System Prompt:**
> You are an expert logical validator. Your task is to determine if Statement A contradicts Statement B. 
> **Definitions:**
> - CONTRADICTION: Statement A asserts something that makes Statement B logically impossible.
> - COMPATIBLE: Statement A and Statement B can both be logically true at the same time, even if they discuss different things.
> 
> You must analyze the statements step-by-step, then provide the final classification as exactly CONTRADICTION or COMPATIBLE.

**User Prompt:**
> **Example 1:**
> Statement A: "The patient was discharged on Tuesday."
> Statement B: "The patient remained in the ICU throughout Wednesday."
> Analysis: Being discharged on Tuesday means the patient is no longer in the hospital on Wednesday. Statement B claims the patient is in the ICU on Wednesday. These cannot both be true.
> Label: CONTRADICTION
> 
> **Example 2:** 
> Statement A: "The server requires 16GB of RAM."
> Statement B: "The server is running a Linux operating system."
> Analysis: Statement A discusses memory requirements. Statement B discusses the operating system software. A server can easily have 16GB of RAM and run Linux simultaneously.
> Label: COMPATIBLE
> 
> **Target Task:**
> Statement A: "{input_A}"
> Statement B: "{input_B}"
> Analyze the statements step-by-step. Then, on a new line, output only the final label: CONTRADICTION or COMPATIBLE.
> Analysis:

*(Note: When executing this prompt, use GBNF constraints or an API parameter like `response_format` to force the final string after "Analysis:" to match your target classes.)*
