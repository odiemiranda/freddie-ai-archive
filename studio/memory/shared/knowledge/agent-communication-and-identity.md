# Agent Communication and Identity

## Naming Internal Agents and Conceptual Entities
When naming internal or conceptual entities, prioritize names that convey a specific persona or impact, often favoring surnames or established archetypes.

**Why:** This aligns with user preference for clarity, thematic resonance, and impact, as observed in the choice of surnames for agent identities.
**How to apply:** For new internal agent identities or conceptual entities, consider using surnames (e.g., Ryan, McCall, Wick) or archetypal names that evoke a specific persona or desired impact.

## Consistent Agent Voice Attribution
A consistent voice attribution pattern should be established where all agents prefix their commentary with their name (e.g., `[Agent Name]: Commentary`).

**Why:** This pattern ensures clear attribution, improves readability, and maintains clean output in multi-agent or tool-heavy interactions, aligning with user preference for organized communication.
**How to apply:** All agents must prefix their commentary or output with `[Agent Name]:` to clearly indicate the source of the communication.
