# OT-0007: Rewrite write-raw.ts to Capture from Claude Code JSONL

## Goal

Replace the current `write-raw.ts` implementation that reads a manually-written markdown file (`--input`) with one that reads Claude Code session JSONL files directly from `~/.claude*/projects/`. This eliminates the lossy summary step -- raw files will contain the actual conversation: user messages, assistant responses, tool calls, tool results, and reasoning chains.

Raw files are now JSON (not markdown). brain.db indexes individual messages, not whole-file chunks.

## Architecture Decisions

### JSONL Source Location

Claude Code stores full conversations as JSONL across multiple `.claude*` directories:
```
~/.claude/projects/{project-slug}/{session-id}.jsonl
~/.claude-personal/projects/{project-slug}/{session-id}.jsonl
~/.claude-team/projects/{project-slug}/{session-id}.jsonl
...any ~/.claude*/projects/ directory
```

The tool must glob `~/.claude*/projects/{project-slug}/` to find session files. This covers `~/.claude/`, `~/.claude-personal/`, `~/.claude-team/`, and any future `.claude-*` directories.

The project slug is the cwd with path separators replaced by `--` (e.g., `d--PROJECTS-freddie-ai` for `d:\PROJECTS\freddie-ai`). Auto-detect picks the most recently modified `.jsonl` file across ALL matched directories. The `--session` flag overrides with a specific session ID.

### JSONL Message Structure

Each line is a JSON object with a `type` field. Key fields per type:

```
type: "user"
  .message.role = "user"
  .message.content = string | ContentBlock[]
  .isMeta = boolean (true for command caveats, skill injections)
  .timestamp = ISO string
  .promptId = string (present on real user-typed messages)

type: "assistant"
  .message.role = "assistant"
  .message.content = ContentBlock[]   (text | tool_use | thinking)
  .timestamp = ISO string

type: "progress"     -- hook execution noise
type: "file-history-snapshot"  -- git state snapshots
type: "system"       -- local commands, hook summaries, turn duration
type: "last-prompt"  -- internal bookkeeping
```

ContentBlock types within assistant messages:
```
{ type: "text", text: "..." }
{ type: "tool_use", id: "...", name: "ToolName", input: { ... } }
{ type: "thinking", thinking: "..." }
```

Tool results appear as user messages with content as array:
```
{ type: "tool_result", tool_use_id: "...", content: "..." }
```

### Capture Rules

| JSONL type | Content block type | Capture? | Notes |
|------------|-------------------|----------|-------|
| `user` (not meta) | text (string) | YES | User-typed messages. Filter out `<command-name>`, `<local-command-stdout>`, `<local-command-caveat>` wrapped content. |
| `user` (meta, list) | text | YES | Skill injections -- these carry the actual skill instructions |
| `assistant` | text | YES | Freddie/agent responses |
| `assistant` | tool_use | YES | Tool name + input args (JSON stringified) |
| `user` (list) | tool_result | YES | Truncate content to 500 chars |
| `assistant` | thinking | YES | Reasoning chains (may be empty string -- skip if empty) |
| `progress` | * | SKIP | Hook execution noise |
| `file-history-snapshot` | * | SKIP | Reconstructable from git |
| `system` | * | SKIP | Boilerplate (local commands, hook summaries, turn durations) |
| `last-prompt` | * | SKIP | Internal bookkeeping |

### Tool Result Truncation

If tool_result content exceeds 500 characters, truncate to first 500 chars and append `...(truncated, {total} chars)`.

### Output Raw File Format

Output is a `.json` file, not markdown. No frontmatter, no fenced blocks.

**Filename:** `vault/studio/memory/raw/{unix-timestamp}-{sessionId}.json`

```json
{
  "ts": 1775007805,
  "date": "2026-04-01",
  "sessionId": "fc959924-...",
  "agents": ["freddie", "mccall", "wick"],
  "skill": "ad-hoc",
  "tags": ["mcp", "notebooklm"],
  "summary": "one-line summary",
  "summaryBody": "Full narrative summary...",
  "messages": [
    {"ts":"...","role":"user","type":"text","content":"..."},
    {"ts":"...","role":"assistant","type":"tool_use","tool":"Bash","args":{}},
    {"ts":"...","role":"tool_result","tool":"Bash","content":"output here"},
    {"ts":"...","role":"assistant","type":"thinking","content":"reasoning..."}
  ]
}
```

### Captured Message Schema

Each object in the `messages` array follows one of these shapes:

```typescript
// User text message
{ ts: string; role: "user"; type: "text"; content: string }

// Assistant text response
{ ts: string; role: "assistant"; type: "text"; content: string }

// Assistant tool call
{ ts: string; role: "assistant"; type: "tool_use"; tool: string; args: Record<string, unknown> }

// Tool result
{ ts: string; role: "tool_result"; tool: string; content: string }

// Thinking block
{ ts: string; role: "assistant"; type: "thinking"; content: string }
```

### brain.db: Per-Message Indexing

`insertRawFile()` creates individual rows per message, not one big chunk:

```
raw_files (1 row per .json file)
  └── raw_chunks (1 row for summaryBody + 1 row per message)
        └── raw_fts (FTS5 index per chunk)
        └── vec_raw (vector embedding per chunk)
```

The summary chunk is its own separate row -- it does NOT duplicate into message rows. `--search-raw` returns specific messages, not whole files.

### 2. `src/libs/raw.ts` -- Update writeRaw()

Modify `RawFileInput` and `writeRaw()` to write JSON instead of markdown.

```typescript
// Types stay the same as current:
```

- Build a plain object with `ts`, `date`, `sessionId`, `agents`, `skill`, `tags`, `summary`, `summaryBody`, `messages`
- Write `JSON.stringify(obj, null, 2)` to the file
- Keep atomic write pattern (tmp + rename)
- Update collision handling for new filename pattern (`{ts}-{sessionId}.json`, collision suffix: `{ts}-{sessionId}-1.json`)

Remove `buildFrontmatter()` -- no longer needed.

Remove `parseFileIdentity()` -- update to handle new filename format if still needed for collision identity.

### 3. `src/libs/brain/raw.ts` -- Per-Message Indexing

**This section has changed from "No Changes Required" to significant changes.**

`insertRawFile()` must handle `.json` files with per-message indexing:

```typescript
async function insertRawFile(filePath: string): Promise<void>
  // 1. Read and JSON.parse the .json file
  // 2. Create parent row in raw_files (ts, date, agents, skill, tags, summary)
  // 3. Create one chunk for summaryBody (the semantic anchor for search)
  //    - Insert into raw_chunks + raw_fts + vec_raw
  //    - This is the "what happened" overview chunk
  // 4. Create one chunk per message in messages[]
  //    - Each message gets its own row in raw_chunks + raw_fts + vec_raw
  //    - Content for FTS/embedding: serialize the message to a search-friendly string
  //      e.g., for tool_use: "tool_use: Bash — git log --oneline"
  //      e.g., for text: the content string directly
  //      e.g., for tool_result: "tool_result: Bash — {truncated content}"
  // 5. The summary chunk is a SEPARATE row -- it does NOT duplicate into message rows
```

**Embedding strategy for messages:**
- `type: "text"` (user or assistant) -- embed the content directly
- `type: "tool_use"` -- embed `"{tool}: {JSON.stringify(args)}"` (or a readable summary)
- `type: "tool_result"` -- embed `"result from {tool}: {content}"`
- `type: "thinking"` -- embed the thinking content directly

This enables `--search-raw` to return specific messages (e.g., "find the Bash command that ran git log") rather than entire session dumps.

## Acceptance Criteria

1. `bun src/tools/write-raw.ts --agents freddie --summary "test"` reads the most recent JSONL from `~/.claude*/projects/` and produces a `.json` raw file in `vault/studio/memory/raw/`
2. The raw file is valid JSON with `ts`, `date`, `sessionId`, `agents`, `skill`, `tags`, `summary`, `summaryBody`, and `messages` fields
3. The `messages` array contains individual message objects, one per captured JSONL event
4. `progress`, `file-history-snapshot`, `system`, and `last-prompt` JSONL types are excluded
5. Tool results are truncated to 500 chars with `...(truncated, N chars)` suffix
6. Empty thinking blocks are excluded
7. User messages wrapped in `<local-command-stdout>`, `<local-command-caveat>`, or `<command-name>` tags are excluded
8. The `--session` flag resolves the correct JSONL file across all `~/.claude*/projects/` dirs
9. Auto-detect (no `--session`) finds the most recent JSONL across all `~/.claude*/projects/` dirs
10. `--summary-file` content populates `summaryBody`; `--summary` populates `summary`
11. brain.db has individual rows per message in `raw_chunks` + `raw_fts` + `vec_raw`
12. brain.db has a separate summary chunk (from `summaryBody`) that does NOT duplicate into message rows
13. `--search-raw` returns specific messages, not whole files
14. The `--input` flag is removed; old callers will get a clear error
15. `/bye` and `/save-raw` skill files are updated to use the new CLI interface
16. The tool does NOT import or depend on any Claude Code internals -- it reads JSONL as plain files
17. Tool registry `toolDef` is updated with the new input schema
18. Raw filename pattern is `{unix-timestamp}-{sessionId}.json`
