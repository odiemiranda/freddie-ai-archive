# OT-0006: Auto-Write Raw Context from Reflect

## Goal

When `reflect.ts` runs without `--raw-ref`, it should auto-generate a raw context file from its own trace data before reflecting, then pass the resulting timestamp into the reflection as `rawRef`. This closes the gap where ad-hoc sessions (not using `/dev-start` or `/dev-end`) miss raw context capture entirely.

## Architecture Decisions

These are final. Do not deviate.

- **Change is scoped to `src/tools/reflect.ts` only.** The CLI/tool layer. Do NOT modify `src/libs/reflect.ts` or `src/libs/raw.ts`.
- **When `--raw-ref` is NOT provided**, reflect auto-generates a raw context file from the trace before calling `reflect()`.
- **When `--raw-ref` IS provided**, skip auto-write entirely. The caller already handled it (e.g., `/dev-end`).
- **No new files.** No new dependencies. Import `writeRaw` from `../libs/raw.js` and `initBrain`/`insertRawFile` from `../libs/brain/index.js`.
- **Raw file gets two sections:** `## Task` and `## Trace` (see spec below).
- **The resulting timestamp is passed to `reflect()` as `rawRef`.**
- **Brain indexing on auto-write:** After `writeRaw()`, call `initBrain()` + `insertRawFile()` to index it, same pattern as `write-raw.ts`. If indexing fails, log to stderr but do NOT block reflection.

## Spec

### What changes in `runReflect()`

Insert auto-raw logic after trace resolution but before the `reflect()` call. Pseudocode:

```
function runReflect(args):
  // ... existing trace resolution (lines 39-45) ...

  let rawRef = args.rawRef

  // Auto-write raw if no rawRef provided
  if (!rawRef):
    rawResult = writeRaw({
      agents: [args.agent],
      skill: args.workflow,
      tags: ["auto-reflect"],
      summary: args.task,
      sections: [
        { name: "Task", content: buildTaskSection(args) },
        { name: "Trace", content: trace }
      ]
    })

    if (rawResult.written):
      rawRef = rawResult.timestamp
      // Index into brain.db — best-effort, don't block reflection
      try:
        initBrain()
        await insertRawFile(rawResult.filePath)
      catch:
        console.error("[reflect] raw indexing failed, continuing")

  // Pass rawRef (auto-generated or caller-provided) into reflect()
  const result = await reflect({ ...args, rawRef, trace })
  // ... rest unchanged ...
```

### `buildTaskSection` helper

A small helper function inside `reflect.ts` (not exported) that formats the Task section:

```
function buildTaskSection(args: ReflectArgs): string
  returns:
    "**Agent:** {agent}\n**Task:** {task}\n**Workflow:** {workflow}\n**Outcome:** {outcome}"
    + "\n**Feedback:** {feedback}" if feedback exists
```

This gives the raw file a structured summary separate from the full trace dump.

### Imports to add

At the top of `src/tools/reflect.ts`, add:

```typescript
import { writeRaw } from "../libs/raw.js";
import { initBrain, insertRawFile } from "../libs/brain/index.js";
```

### CLI and toolDef changes

None. The `--raw-ref` flag remains optional in both CLI and `toolDef`. The only behavioral change is what happens when it is omitted.

## Acceptance Criteria

Test manually -- no test framework (Bun test is broken on Windows).

1. **Reflect without `--raw-ref`** -- run `bun src/tools/reflect.ts --agent tala --task "test task" --workflow create-track --outcome success --trace "Did some things, got results"`. Verify a new file appears in `vault/studio/memory/raw/` with `## Task` and `## Trace` sections. Verify the frontmatter has `agents: ["tala"]`, `skill: "create-track"`, `tags: ["auto-reflect"]`.
2. **Reflect with `--raw-ref`** -- run with `--raw-ref 1234567890`. Verify NO new raw file is created. The provided rawRef is passed through to `reflect()`.
3. **Raw file content** -- the Task section contains agent, task, workflow, outcome formatted as bold key-value pairs. The Trace section contains the full trace text verbatim.
4. **Brain indexing** -- after auto-write, verify the raw file was indexed into brain.db (check via `bun src/tools/brain.ts --search-raw "test task"`).
5. **Brain indexing failure is non-fatal** -- if brain.db is locked or broken, reflect still completes and prints its normal JSON output. The only evidence of failure is a stderr line.
6. **Feedback field** -- when `--feedback` is provided without `--raw-ref`, the Task section includes the feedback line.
7. **Empty trace rejection** -- reflect still fails with "--trace or --traceFile required" when no trace is provided, regardless of rawRef.
8. **toolDef path** -- when called via registry (`toolDef.execute`), the same auto-raw behavior applies (since both paths go through `runReflect`).
