## Decisions

- **Style block formula locked:** Style = sonic palette + instrument behavior + dynamic control. NO structure tags (those go in Lyrics only). `#suno`
- **Genre must be rhythm template:** "Jazz fusion" not "groove pop" or "chillpop" — vague genres don't give Suno a strong rhythm foundation. `#suno`
- **Neutral instrument labels:** "harp" not "celtic harp" to prevent folk genre bleed. `#suno`
- **Bracket tags 6-10 words:** With positional cues for support instruments. 3-5 too short, 12+ risk being sung. `#suno`
- **Musical notation rejected:** Tested A/B/C — descriptive (100%), register+rhythm (80%), full notation (70%). Note names and voicings degrade output. `#suno`
- **Three-tier memory system adopted:** inbox → knowledge → archive replaces flat files. Agents save to inbox, read from knowledge, consolidation promotes. `#memory`
- **Audio file slug format:** `<unixtimestamp>-<slug>.mp3`, card .md file without timestamp prefix. `#music-cards`
- - **Gemini SDK over CLI — done:** @google/genai replaces gemini-proxy.ts. Phase 1 + Phase 4 completed same day. `#architecture` `#sdk`
- **Echo owns discovery:** Echo handles reference loading and prompt assembly. `tools/gemini.ts` is just SDK call + system prompt. Smarter agent, dumber tool. `#architecture`
- **Auto-memory scope locked:** Auto-dream writes user/feedback only. Production rules stay in vault knowledge. Prevents knowledge pollution. `#memory`
- **Usage tracking auto-wired:** Every `generateText()` call logs to JSONL. Caller auto-derived from `--system-file` name. `#tracking`

## Learned

- **Dangerous vibe words in Suno:** "lazy" → "laid-back", "distant" → "background/spacious", "subtle melodic movement" → causes noodling. `#suno`
- **Support instrument hierarchy is critical:** Without subordination in BOTH Style and Lyrics, Suno gives all instruments equal presence and they compete. `#suno`
- **"plucks" triggers aggressive attack:** Use "smooth melodic phrase" for shamisen instead. `#suno`
- **Windows file-lock with yt-dlp:** Can't rename .webm.part during postprocessing. Fix: two-step download (yt-dlp raw → ffmpeg trim+convert). `#tooling`
- **Windows renameSync silently fails:** Use copyFileSync + unlinkSync instead for cross-drive operations. `#tooling`
- **librosa half-tempo issue:** Lo-fi tracks report doubled BPM (172 instead of 86). Correct by halving if >140 and half is in 55-140 range. `#audio-analysis`
- **Exclude list breadth matters:** 5-6 items too narrow, instruments bleed in. Clean tracks need 10-18 excludes. `#suno`
- - **Gemini API rate limits:** Free tier = 10 RPM / 250K TPM / 50 RPD. Paid = 2000 RPM / 4M TPM / unlimited RPD. `#research`
- **Auto-dream can pollute vault knowledge:** Claude Code's built-in memory system writes directly without review. Must scope it to user/feedback only, or production rules get duplicated and can drift. `#memory`
- **Smarter agent, dumber tool pattern:** Moving business logic (discovery, routing) into the agent file and keeping the CLI tool as a thin SDK wrapper makes both easier to maintain and test. `#architecture`
