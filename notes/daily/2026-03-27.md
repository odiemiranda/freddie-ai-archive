## Decisions

- Typed brackets are the SINGLE unified pattern for ALL tracks — no more dual strategy
- Per-section `[Instrument:]` tags replace Exclude lists — positive constraints beat negative
- Style block ~75-120 chars for everything — all detail in Lyrics typed brackets
- `[Callback:]` is musical memory within a generation, not an Extend feature
- Atmosphere tags need more testing — partial results, not blocking
- Community reference data goes in vault reference files, not in agent files (keeps agents lean)

## Learned

- Typed brackets work for ALL tracks — not just vocal. Instrumental lo-fi tested, same quality as ensemble pipes
- Suno reads Lyrics typed brackets more carefully than Style descriptors for per-section control
- Ultra-lean Style (~75-120 chars) + tag-rich Lyrics outperforms verbose Style for all track types
- `[modulate up a key]` must go BEFORE lyrics — position matters. Numeric values ignored (always 1 step)
- `[Buildup]` is stackable — multiple lines = longer/more intense ramp
- `~word~` elongates syllables, `word-` creates smooth musical cut off, `"word"` triggers spoken delivery
- `[crowd sings]` triggers live concert feel, `[Callback: Chorus melody]` reuses prior section's musical material
- `[angry verse]` / `[hopeful chorus]` fused lowercase emotion+section works
- ALL CAPS: max 1-3 words per section for impact
- Scream/growl: stretch vowels (AAAAAH not AH) for harder hit
- Atmosphere tags ([Rain], [Thunder]) produce brief sounds but not sustained — needs more testing
- When every section has [Instrument:] tags, Exclude Style can be empty — positive beats negative
- Era anchors (60s, 80s) in pipes help genre accuracy
- Bar counts work in any pipe element, not just [Buildup]
